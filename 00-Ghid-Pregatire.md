## Principal Engineer vs Senior Engineer - Diferențe cheie

### Ce se așteaptă de la un Principal Engineer

Un **Principal Engineer** nu este doar un Senior mai experimentat. Diferența este calitativă, nu cantitativă:

| Dimensiune | Senior Engineer | Principal Engineer |
|------------|----------------|-------------------|
| **Scop** | Echipa proprie | Organizație / multiple echipe |
| **Impact** | Livrează funcționalități complexe | Definește direcția tehnică |
| **Decizie** | Alege soluția pentru task | Definește arhitectura pe termen lung |
| **Influență** | Prin cod și code review | Prin viziune tehnică, mentorat, RFCs |
| **Autonomie** | Primește probleme, livrează soluții | Identifică problemele înainte să apară |
| **Comunicare** | Cu echipa | Cu stakeholderi, management, alte echipe |
| **Mentoring** | Ajută colegii din echipă | Ridică nivelul tehnic al organizației |

### Cum să te comporți în interviu la nivel Principal

1. **Gândește la scară largă** - Nu vorbi doar despre cum rezolvi un bug, ci despre cum previi categorii întregi de buguri
2. **Arată trade-off-uri** - Nu există soluții perfecte. Arată că înțelegi compromisurile fiecărei decizii
3. **Comunică viziune** - Explică "de ce" înainte de "cum"
4. **Demonstrează leadership tehnic** - Dă exemple de decizii care au influențat mai multe echipe
5. **Fii pragmatic** - Cele mai bune soluții sunt cele care funcționează, nu cele mai elegante
6. **Discută experiența** - La nivel Principal, experiența reală contează mai mult decât teoria

## Sfaturi practice pentru interviu

### Pregătire

- **Recitește CV-ul** - Fii pregătit să discuți orice proiect menționat
- **Pregătește 3-5 "war stories"** - Proiecte complexe, decizii dificile, eșecuri din care ai învățat
- **Exersează explicarea conceptelor** - Poți explica Change Detection unui junior?
- **Cunoaște stack-ul companiei** - Adaptează exemplele la tehnologiile lor

### În timpul interviului

- **Începe cu contextul** - Înainte de a răspunde, clarifică problema
- **Structurează răspunsurile** - Folosește metoda STAR pentru întrebări comportamentale
- **Scrie cod curat** - Chiar și la whiteboard, arată că îți pasă de calitate
- **Pune întrebări** - Un Principal Engineer întreabă "de ce?", nu doar "cum?"
- **Admite ce nu știi** - E mai bine să spui "Nu știu, dar aș investiga X" decât să inventezi

### Semnale de nivel Principal

Intervievatorii caută aceste semnale:

- **Ownership** - "Am propus și implementat..."
- **Impact la scară** - "Am migrat 15 echipe la..."
- **Viziune pe termen lung** - "Am anticipat că vom avea nevoie de..."
- **Mentoring** - "Am ghidat echipa prin..."
- **Maturitate** - "Am greșit când... și am învățat că..."

## Versiuni Angular acoperite (până la ultima: **v21**)

Acest material este aliniat la **ultima versiune Angular 21** și la recomandările oficiale de pe [angular.dev](https://angular.dev). Acoperă **Angular 17–21** (2024–2026):

| Versiune | Data | Noutăți cheie |
|----------|------|---------------|
| **Angular 17** | Nov 2023 | Signals (developer preview), `@defer`, control flow nou (`@if`, `@for`, `@switch`) |
| **Angular 18** | Mai 2024 | Zoneless experimental, Material 3 stabil |
| **Angular 19** | Nov 2024 | Standalone by default, LinkedSignal, Resource API, incremental hydration |
| **Angular 20** | Mai 2025 | **Signals API stabil** (effect, linkedSignal, toSignal), Zoneless developer preview, template literals în templates, `**` operator, `in` keyword, host binding type checking, HttpResource, HammerJS deprecat |
| **Angular 21** | Nov 2025 | **Signal Forms** (experimental), **Zoneless default** pentru apps noi, **Vitest** default test runner, **Angular Aria** (headless accessible components), **MCP Server** (7 tools pentru AI), Generic SimpleChanges, HttpClient provided by default |

### Ce e HOT pentru interviu (Angular 20–21)
- **Signal Forms** — noul API de formulare bazat pe signals (`form()`, `FormField`, `submit()`)
- **Zoneless by default** — `provideZonelessChangeDetection()` (default pentru app-uri noi în v21)
- **Vitest** — înlocuiește Karma/Jest ca default test runner
- **Angular Aria** — componente headless accesibile (Developer Preview)
- **Angular MCP Server** — AI poate interacționa cu Angular CLI (7 tools)
- **Best practices oficiale pentru AI** — [angular.dev/ai](https://angular.dev/ai), [develop-with-ai](https://angular.dev/ai/develop-with-ai): `best-practices.md`, `llms.txt`, reguli per IDE (Cursor, Copilot, etc.)

## Recomandări oficiale Angular (angular.dev) — utile la interviu

Conform [angular.dev/ai/develop-with-ai](https://angular.dev/ai/develop-with-ai) și Style Guide:

- **Standalone** — folosește întotdeauna standalone components; în **v20+ nu seta** `standalone: true` în decorator (e default).
- **Signals** — pentru state management; `input()` / `output()` în loc de `@Input()` / `@Output()`.
- **Host** — nu folosi `@HostBinding` / `@HostListener`; pune bindings în obiectul `host` al decoratorului.
- **Templates** — control flow nativ: `@if`, `@for`, `@switch`; nu `ngClass`/`ngStyle` — folosește `[class]` / `[style]`.
- **Imagini** — `NgOptimizedImage` pentru imagini statice (nu pentru base64 inline).
- **Servicii** — `inject()` în loc de constructor injection; `providedIn: 'root'` pentru singleton.
- **Change detection** — `ChangeDetectionStrategy.OnPush` în componente.

Documentație AI: [angular.dev/ai](https://angular.dev/ai), [angular.dev/ai/develop-with-ai](https://angular.dev/ai/develop-with-ai).
